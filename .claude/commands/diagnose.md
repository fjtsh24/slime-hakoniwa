# バグ診断フロー

**修正コードを書く前に、必ずこのフローを完了すること。**

## Step 1: 関連ファイルを全て読む

エラーに関連する以下を確認する：
- エラーメッセージ・スタックトレースが示すファイルと行番号
- 呼び出し元のコンポーネント・関数
- 関連する設定ファイル（`netlify.toml`, `tsconfig.json`, `firestore.rules` 等）
- 環境変数の使用箇所

## Step 2: 問題点を全て列挙する

修正に着手する前に、発見した問題を全て箇条書きで提示する：

```
【診断結果】
- 問題1: ○○ファイルの△△行で□□が未設定
- 問題2: ◇◇の依存関係が不足
- 問題3: ...

【根本原因】
（なぜこの問題が起きているか）

【修正方針】（ユーザーの承認を得てから実施）
1. ...
2. ...
```

## Step 3: ユーザーの承認を得てから修正する

診断結果と修正方針を提示し、承認を得てから実装する。

## Step 4: 修正後の確認

- ローカルでビルド・テストを実行して修正を確認する
- 関連する他のコンポーネントへの影響がないか確認する
- 必要に応じてコミット・PR作成まで行う

## よくあるパターン

| 症状 | 確認すべき箇所 |
|------|--------------|
| 401 / 認証エラー | IDトークン送信コード・`verifyIdToken` ・Firebase Admin SDK 初期化 |
| ビルドエラー (`not found`) | `package.json` の依存関係・ビルドコマンド順序 |
| Firestore エラー | `firestore.indexes.json`・`firestore.rules`・クエリのフィールド一致 |
| 型エラー (TypeScript) | `tsconfig.json`・インポートパス・型定義ファイル |
| デプロイ失敗 | `netlify.toml`・環境変数設定・secrets scanning |
