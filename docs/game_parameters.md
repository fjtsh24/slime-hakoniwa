## 1. ターン進行パラメータ

### 1-1. ターン間隔: 3600秒（1時間）の評価

**判定: 「まったりゲーム」として適切**

| 項目 | 値 |
|------|-----|
| ターン間隔 | 3600秒（1時間） |
| 1日のターン数 | 24ターン |
| 1週間のターン数 | 168ターン |

#### 評価根拠

「箱庭諸島」的な「行動予約→放置→結果確認」のサイクルに1時間ターンは適切なテンポ感を持つ。

- **プレイスタイルとの一致**: 1日数回チェックする「ながら育成」と相性がよい。毎日朝・昼・晩の3回チェックするだけで3ターン分の結果を楽しめる。
- **行動予約のバッファ**: 最大10件の予約（実装済み）があれば、10時間分（10ターン分）先まで予約可能。就寝前に翌朝分まで予約しておける。
- **進化タイムラインとの整合性**: 後述の計算により、第1進化まで概ね5〜14日で到達でき、飽きる前に変化を体験できる（長すぎず短すぎない）。
- **リアルタイム感**: 1時間ターンは「時計と同期している感覚」をプレイヤーに与え、現実の時間とゲーム時間が連動する体験を生む。

#### 注意事項

- `turnIntervalSec` のデフォルト値は `functions/src/scripts/seed.ts` で **3600** に設定済み。
- 変更する場合は必ずA1/Funに確認を取ること（CLAUDE.md ルール）。

## 2. hunger（満腹度）設計

### 2-1. 現在の実装確認

`functions/src/scheduled/turnProcessor.ts` の実装を精査した結果、以下を確認した。

#### hunger 消費（毎ターン固定）

```
毎ターン終了時: hunger -= 5（下限0）
```

食事アクション・移動アクション・休息アクションを実行したターンも、
**ターン処理の最後に必ず hunger -5 が適用される**（予約アクション実行後に減算）。

#### 自律行動時の hunger 別挙動

| hunger 範囲 | 自律行動 |
|------------|---------|
| 50以上 | 元気に歩き回る（HP変化なし） |
| 20〜49 | 休息してHP微回復（maxHP × 5%） |
| 0〜19 | 弱って動けない（HP回復なし） |

#### 食事アクションによる hunger 回復

```
eat アクション実行時: hunger += 30（上限100）
```

### 2-2. 食料マスタからの hunger 回復計算

`shared/data/foods.ts` を確認した。食料データには `statDeltas` が定義されているが、**hunger の直接回復量は食料ごとに異なるのではなく、eat アクション実行で一律 +30** の実装になっている（食料の種類に関係なく固定値）。

| 操作 | hunger 変化 |
|------|------------|
| ターン経過（毎回） | -5 |
| eat アクション | +30 |
| rest アクション（予約） | +10（HP微回復も発生） |

### 2-3. hunger バランス試算

#### 基本スライム（slime-001）初期 hunger: 80

**何もしない場合（食事予約なし・自律行動のみ）の hunger 推移:**

| 経過ターン数 | hunger | 状態 |
|-------------|--------|------|
| 開始時 | 80 | 元気（歩き回る） |
| 6ターン後（6時間） | 50 | 元気（歩き回る） |
| 7ターン後（7時間） | 45 | 休息モード（HP微回復） |
| 12ターン後（12時間） | 20 | 休息モード（HP微回復） |
| 13ターン後（13時間） | 15 | 弱って動けない |
| 16ターン後（16時間） | 0 | 空腹状態が継続 |

**「空腹で動けない」状態（hunger < 20）になるまでの日数:**

```
初期 hunger 80 → hunger 20 になるまで: (80 - 20) / 5 = 12ターン = 12時間（0.5日）
初期 hunger 80 → hunger 0 になるまで: 80 / 5 = 16ターン = 16時間（約0.67日）
```

**評価**: 何も食事をしないと約半日でスライムが弱ってしまう。1日最低1〜2回の食事予約を促す適度な管理負荷であり、「まったりゲーム」の範囲内で適切。毎日1回（24時間に1回）の食事では hunger は -5×24+30 = -90 となり長期維持は困難。**適切な関与頻度は「1日2〜3回の食事予約」**。

#### 食事1回あたりの hunger サイクル（最適ペース）

| 食事間隔 | ターン数 | 最低 hunger |
|----------|---------|------------|
| 6ターン（6時間）に1回 | 6 | 80→50→80（+30-5×6=0、維持）|
| 5ターン（5時間）に1回 | 5 | 80→55→85（余裕あり） |
| 7ターン（7時間）に1回 | 7 | 80→45→75（自律休息発生） |

**推奨サイクル**: 6ターン（6時間）に1回食事を予約することで、hunger 50以上を維持して常に「元気」状態を保てる。

## 3. 進化条件バランス評価

### 3-1. 現在の進化条件（slime-001 → 第1進化）

```
条件: exp >= 100 かつ 対応属性の種族値 >= 0.5
例: slime-001 → slime-002（ファイアスライム）
    必要: exp >= 100 AND racialValues.fire >= 0.5
```

### 3-2. 食料別EXP付与量

| 食料 | EXP付与 | EXP効率 |
|------|---------|---------|
| 野草（food-plant-001） | 0 | なし |
| 薬草（food-plant-002） | 0 | なし |
| 干し肉（food-human-001） | 0 | なし |
| スライムの欠片（food-slime-001） | 0 | なし |
| スライムコア（food-slime-002） | 0 | なし |
| 川魚（food-fish-001） | 0 | なし |
| プリズムスライムゼリー（food-slime-003） | 5 | 低 |
| 深海魚（food-fish-002） | 5 | 低 |
| 魔法のパン（food-human-002） | 8 | 中 |
| 世界樹の葉（food-plant-003） | 10 | 高 |
| 霊魂の結晶（food-spirit-002） | 15 | 最高 |

### 3-3. 第1進化（EXP 100）への到達日数試算

**前提**: 1日の食事予約回数 = 4回（6時間に1回）、ターン間隔 = 1時間

| 使用食料 | 1回のEXP | 1日のEXP（4回） | 到達日数 |
|---------|---------|----------------|---------|
| 世界樹の葉（EXP 10） | 10 | 40 | 約2.5日 |
| 霊魂の結晶（EXP 15） | 15 | 60 | 約1.7日 |
| 魔法のパン（EXP 8） | 8 | 32 | 約3.1日 |
| プリズムスライムゼリー（EXP 5） | 5 | 20 | 5日 |
| EXP 0の食料のみ | 0 | 0 | 到達不可 |
| ランダム混合（平均EXP 4と仮定） | 4 | 16 | 約6.3日 |

**評価**: EXP条件は5〜14日程度で達成可能な設計になっている。まったりゲームとして「1〜2週間で最初の進化体験」は適切。ただし、**EXPを与えない食料が多い**（8/11種がEXP 0）ため、「どの食料を選べばよいか」の判断がゲームプレイの重要要素になる。この点はチュートリアル設計で誘導が必要。

### 3-4. 種族値 0.5 への到達試算（進化のもう一つの条件）

例：fire 種族値 0.5 を達成するために火属性タイルへの移動を活用する場合。

- タイルの fire 属性 1.0 のタイルへ移動: racialValues.fire += 0.1（1回）
- 5回の移動で fire = 0.5 に到達（5ターン = 5時間）

ただし、タイル属性は 0.0〜1.0 の値を持ち、実際の箱庭マップでの属性分布によって難易度が変わる。現在の seed データでは 100 タイルにランダム属性が付与されている。

**評価**: EXP条件（数日〜2週間）のほうが達成に時間がかかるため、**実質的には EXP が進化のボトルネック**になる。種族値条件は「属性特化の方向性決定」として機能しており、バランスは適切。

### 3-5. 第2進化（EXP 300）への到達日数試算

第2進化は Phase 4 スコープだが、将来のパラメータ設計の参考として試算する。

| 使用食料 | 1日のEXP（4回） | 第1進化後の追加EXP（300-100=200） | 追加日数 |
|---------|----------------|----------------------------------|---------|
| 世界樹の葉（EXP 10） | 40 | 200 | 5日 |
| 霊魂の結晶（EXP 15） | 60 | 200 | 約3.3日 |
| ランダム混合（平均EXP 4） | 16 | 200 | 12.5日 |

**評価**: 第1進化から第2進化まで数日〜2週間の追加プレイが必要。第1進化後に「さらに先を目指す」モチベーションを維持するため、進化後スライムにも固有の楽しさ（新しいスキル・外見変化・ステータス特徴の強調）を用意することが重要（Phase 4 での実装で対応）。

## 4. 初期スライム付与設計（Phase 3 実装方針）

### 4-1. 付与タイミングと方法の決定

**決定事項: プレイヤーが「はじめてのスライムを迎える」ボタンを押したとき**

#### 選択肢の比較

| 方式 | 概要 | 評価 |
|------|------|------|
| **採用: ボタン起点** | GamePage でスライム0体のときにウェルカムカードを表示し、ボタン押下で POST /api/slimes/initial を呼ぶ | ゲーム体験として自然。「迎える」という能動性がプレイヤーの愛着を高める |
| 不採用: Auth Trigger 自動付与 | ユーザー登録時に Cloud Function が自動でスライムを生成 | 技術的にはシンプルだが、「気づいたらいた」という受動体験になる。スライムへの愛着が薄くなる |
| 不採用: SetupPage での即時付与 | マップ初期化時に合わせて付与 | SetupPage はすでにローディング状態であり、ここにスライム付与を重ねると UI が複雑になる |

#### 採用方式の詳細

```
エンドポイント: POST /api/slimes/initial
認証: Firebase IDトークン必須
冪等性: 既にスライムを持つユーザーは 409 Conflict を返す
```

### 4-2. 初期スペック

```
speciesId : "slime-001"（基本スライム）
name      : "はじめてのスライム"
tileX     : 0
tileY     : 0
worldId   : "world-default"（ユーザーのマップに対応するワールドID）
stats     : SlimeSpecies.baseStats をそのまま使用
  hp     : 50
  atk    : 10
  def    : 10
  spd    : 10
  exp    : 0
  hunger : 80
racialValues: 全て 0.0 で初期化
isWild  : false
ownerUid: ログインユーザーのUID
```

#### 初期スペック採用理由

- **基本スライム（slime-001）から始める理由**: 4方向の進化ルートが全て開かれており、プレイヤーが「どの属性に育てるか」を自分で決める楽しみがある。進化選択の自由度がゲームの中核体験のひとつ。
- **hunger 80 の初期値**: 初日に食事をしなくてもすぐに空腹にならない余裕があり、チュートリアルなしでも数時間は元気な状態を維持できる。
- **name "はじめてのスライム"**: 変更不可・固定名称。後のアップデートで名前変更機能を追加する余地を残しつつ、初回体験では世界観に合う名前をデフォルトで提供する。

### 4-3. UI設計（GamePage ウェルカムカード）

GamePage でスライムが0体のとき、スライム一覧の代わりに以下のウェルカムカードを表示する。

```
┌─────────────────────────────────────────────────────────┐
│  🌿 はじめての箱庭へようこそ                              │
│                                                         │
│  まだスライムがいません。                                  │
│  あなたの最初のスライムを迎えてみましょう。                  │
│                                                         │
│  スライムは毎時間行動します。                              │
│  食事や移動を予約して、一緒に育てていきましょう。            │
│                                                         │
│  ┌─────────────────────────────────┐                    │
│  │  はじめてのスライムを迎える      │                    │
│  └─────────────────────────────────┘                    │
└─────────────────────────────────────────────────────────┘
```

#### UI仕様

| 要素 | 内容 |
|------|------|
| 表示条件 | `slimes.length === 0` かつ `userProfile` が存在する |
| ボタン | 「はじめてのスライムを迎える」（クリックで POST /api/slimes/initial） |
| ボタン連打防止 | 送信中は `disabled` 状態にし、ローディングスピナーを表示 |
| 成功時 | スライム一覧が表示され、ウェルカムカードが消える |
| エラー時 | 「スライムの呼び出しに失敗しました。もう一度お試しください。」をインラインで表示 |
| 既にスライムがいる場合（409） | 「すでにスライムがいます」のメッセージを表示し、ページをリロード |

#### ゲームデザイン上の留意点

- このカードがPhase 2で指摘したH-2（スライム0体状態の体験設計）の解決策となる。
- ウェルカムカードはonboarding演出として機能し、ゲームへの入口となる体験として設計する。
- スライムの種族選択制（Phase 2レビュー4-5で提案）はPhase 4以降に延期し、Phase 3では「基本スライム1体からスタート」を固定とする。理由: まず「育てる楽しさ」の基本サイクルを体験させてから選択肢を広げるほうが、初回体験の複雑さを抑えられる。

## 5. 食事アクション体験設計（Phase 3）

### 5-1. 食料の入手方法

**Phase 3 スコープ外**: 食料の入手・在庫管理は実装しない。
マスタデータ（`shared/data/foods.ts`）から直接選択する形式を採用する。

```
食料選択UI: 全11種の食料をリスト表示
            各食料の statDeltas・racialDeltas を表示
            プレイヤーがいずれかを選んで食事アクションを予約
```

この設計により、Phase 3では「どの食料を食べさせるかの戦略」に集中できる。食料入手の仕組み（収集・購入・交換等）はPhase 6以降の課題とする。

### 5-2. 食事1回あたりの体験フロー

```
[1] プレイヤーが食料を選択して「eat」アクションを予約
    → POST /api/reservations
       { slimeId, actionType: "eat", actionData: { foodId }, turnNumber }

[2] ターン進行（1時間後）
    → Cloud Functions が processSlimeTurn を実行
    → eat アクション処理:
       - statDeltas を stats に加算（hp/atk/def/spd/exp）
       - hunger += 30（上限100）
       - racialDeltas を racialValues に加算
       - スキル付与確率チェック（skillGrantProb）
    → hunger -= 5（毎ターン減算）
    → 進化条件チェック

[3] プレイヤーがターンログで結果確認
    → GET /api/worlds/:worldId/turnLogs（Phase 3で実装）
    → 「はじめてのスライムが世界樹の葉を食べた。EXP +10 hunger 80→105→100」
       のようなログを表示
```

### 5-3. 食料別の育成戦略ガイドライン（A1 設計方針）

Phase 3 で食料選択 UI を実装する際、各食料の「育てる方向性」が伝わるように表示設計すること。

| 育成目標 | 推奨食料 |
|---------|---------|
| 素早く進化させたい | 世界樹の葉（EXP 10）、霊魂の結晶（EXP 15） |
| ファイアスライムに進化させたい | 食事 + fire属性タイルへの移動予約を組み合わせる |
| HPを強化したい | スライムコア（HP +5）、世界樹の葉（HP +5）、魔獣の心臓（HP +5） |
| スキルを習得させたい | 魔獣の心臓（確率 12%）、霊魂の結晶（確率 15%）、プリズムスライムゼリー（確率 10%）|

### 6-2. hunger ゲームループの初回体験

初回プレイの数ターンで以下を自然に体験させること:

1. スライムの hunger が下がっているのを確認
2. 食事を予約する
3. ターンが進んで hunger が回復し、EXPが増えているのを確認
4. 「予約→放置→結果確認」のサイクルに慣れる

GamePage でのスライムステータス表示には **現在の hunger 値と次ターン後の予測 hunger 値** を表示することを推奨する。

## 7. まとめ：主要設計決定事項

| 項目 | 決定内容 |
|------|---------|
| ターン間隔 | 3600秒（1時間）を維持。変更不要 |
| 1日の推奨食事回数 | 4回（6時間に1回）で hunger 50以上を維持 |
